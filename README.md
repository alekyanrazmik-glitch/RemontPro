# RemontPro — сайт-платформа ремонтной студии

Современный многостраничный сайт ремонтной студии **RemontPro** (ремонт квартир,
домов и коммерческих помещений под ключ в **Геленджике** и **Новороссийске**).

Это не типовой лендинг, а сайт-платформа с публичной частью и **демо-админкой**
для управления портфолио и видео. Сделан на чистом HTML/CSS/JS — без React и
сборщиков. Полностью работает на **GitHub Pages**.

> Концепция: «ремонт как управляемый проект» — этапы, смета, фотофиксация,
> контроль сроков. Визуальный стиль — sage-pine green, латунь/золото и тёплый
> кремовый (cream/warm white).

## Как данные попадают на сайт

Публичная часть читает JSON через `fetch` со стратегией:

1. **localStorage** — если в этом браузере есть правки из админки, показываются они
   (удобно для предпросмотра своих изменений);
2. **JSON-файл** — `assets/data/projects.json` или `videos.json`;
3. **demo-fallback** — встроенные демо-данные, если `fetch` недоступен
   (например, при открытии через `file://`).

Чтобы изменения увидели **все посетители**, нужно обновить JSON-файлы в репозитории
(см. ниже про экспорт).

### 2. Как добавить проект через админку
1. Откройте `admin/` (на сайте — ссылка «Админ-панель» в подвале).
2. Нажмите **«Добавить проект»**.
3. Заполните поля: название, город, площадь, тип ремонта, срок, бюджет, статус,
   описание, задача, что сделали.
4. Вставьте ссылку на **главное фото** и при необходимости несколько ссылок в
   **галерею** (кнопка «＋ Добавить фото»).
5. Вставьте ссылку на **YouTube** (поддерживаются `youtube.com/watch?v=…` и
   `youtu.be/…`) — появится превью.
6. При желании добавьте фото **«До»** и **«После»** для слайдера.
7. Нажмите **«Сохранить проект»** — он сохранится в `localStorage`.

> ⚠️ Загрузка файлов изображений на сервер в демо-режиме недоступна. Залейте фото
> в `assets/images/` (и закоммитьте) или на внешний хостинг и вставляйте ссылки.

### 3. Как экспортировать JSON
В админке (Дашборд, «Проекты» или «Видео») нажмите **«Экспорт projects.json»** /
**«Экспорт videos.json»**. Браузер скачает актуальный JSON со всеми вашими правками.

### 4. Как заменить projects.json
Положите скачанный файл в `assets/data/projects.json` (заменив старый) и закоммитьте:
```bash
git add assets/data/projects.json
git commit -m "Обновлены проекты портфолио"
git push
```
После деплоя GitHub Pages новые проекты увидят все посетители. Аналогично —
`videos.json`.

> Кнопка **«Сбросить к файлам»** на дашборде очищает localStorage и заново
> загружает данные из JSON-файлов.

### 5. Как добавить YouTube-видео
1. Откройте `admin/videos.html`.
2. Заполните название, вставьте ссылку YouTube, укажите категорию и описание.
3. Нажмите **«Добавить видео»** → затем **«Экспорт videos.json»** и замените файл
   в `assets/data/videos.json`.

Ссылка автоматически конвертируется в `embed`, на сайте видео выводится через
`iframe` с **lazy-загрузкой** (подгружается только по клику на превью).

### 6. Как подключить настоящий backend в будущем
Текущая админка — **демонстрационная**: данные хранятся в `localStorage` браузера
и не защищены. Это нормально для GitHub Pages, но для продакшена нужен сервер.
Варианты развития:

- **Headless CMS** (Strapi, Directus, Sanity, Contentful) — публичная часть
  забирает данные через их API вместо локального JSON.
- **Netlify CMS / Decap CMS** — правки коммитятся прямо в репозиторий через Git.
- **Свой backend** (Node.js/PHP/Python) с авторизацией и базой данных: замените в
  `assets/js/main.js` источник данных в функции `loadData()` на запрос к вашему API,
  а формы заявок (`.js-lead`) подключите к эндпоинту или сервису форм (Formspree).
- **Хранение изображений** — S3-совместимое объектное хранилище или CDN.

При переходе на backend логика рендера на фронте остаётся прежней — меняется только
источник данных.

## Публикация на GitHub Pages

1. Запушьте репозиторий на GitHub.
2. **Settings → Pages → Source: Deploy from a branch**, ветка `main`, папка `/(root)`.
3. Сайт будет доступен по адресу `https://<username>.github.io/<repo>/`.

Все ссылки в проекте **относительные** — сайт работает и в корне домена, и в
подпапке `/<repo>/`. Абсолютные пути не используются.

## SEO

- Уникальные `title` и `description` на каждой странице, один `<h1>`, корректная
  иерархия `h2/h3`.
- `canonical`, Open Graph и Twitter Card.
- Микроразметка schema.org: `HomeAndConstructionBusiness` (главная и страницы
  городов), `FAQPage` (FAQ), `VideoObject` (добавляется на странице проекта при
  наличии видео).
- `sitemap.xml` и `robots.txt` (админка закрыта от индексации).
